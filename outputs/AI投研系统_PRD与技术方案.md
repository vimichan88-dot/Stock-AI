# AI 投研系统 PRD 与技术方案

版本：v1.0  
日期：2026-06-29  
定位：个人专用、中文、A 股和港股为主，美股及全球宏观辅助的 AI 投研日报系统

## 1. 产品目标

本系统目标是每天自动抓取全球资本市场、上市公司公告、宏观数据、AI 行业、新能源行业、行情与资金流信息，经 AI 分析后生成中文机构投研风格报告，并通过私人网页、微信提醒和邮件备份发送给用户。

系统不是普通新闻摘要器，而是个人 AI 投研助理。它需要回答：

- 今天真正重要的事件是什么
- 市场为什么这样交易
- 哪些趋势正在形成
- 哪些公司和行业受益
- 哪些风险正在上升
- 未来 1 周、1 个月、3 个月、1 年应该关注什么
- 当前投资上应如何行动

## 2. 用户与使用场景

目标用户：个人投资者，自用。

语言：中文。

主要关注市场：

- A 股：核心市场
- 港股：核心市场
- 美股：重点关注科技、AI、半导体、能源和宏观影响
- 全球市场：作为宏观、资金流和风险偏好辅助输入

风险偏好：中等偏积极，但必须标注成功率、置信度、风险等级和失效条件。

## 3. 最终运行方式

第一版采用轻量云端自动运行方案：

```text
GitHub Actions 定时运行
  ↓
抓取免费公开数据
  ↓
调用 OpenAI API 生成分析
  ↓
生成静态网页
  ↓
发布到 GitHub Pages
  ↓
PushPlus 微信提醒 + 邮件备份
```

第一版不要求用户购买云服务器，不要求本地电脑长期开机。

## 4. 用户需要配置的最少内容

用户需要准备：

| 配置项 | 用途 | 是否必须 |
|---|---|---|
| GitHub 账号 | 代码仓库、定时任务、网页托管 | 必须 |
| OpenAI API Key | AI 分析与报告生成 | 必须 |
| PushPlus Token | 微信个人提醒 | 推荐 |
| Server 酱 SendKey | 微信提醒备用方案 | 可选 |
| 接收邮箱 | 邮件备份 | 推荐 |
| 邮件服务密钥或 SMTP 授权码 | 自动发送邮件 | 可选 |
| 私人访问 token | 保护网页访问 | 必须 |

后续配置集中放在 GitHub Secrets 中，避免明文暴露。

## 5. 每日运行频率

系统每天自动生成三类报告。

| 时间 | 报告 | 字数建议 | 核心目标 |
|---|---|---:|---|
| A 股开盘前 | 盘前投研日报 | 5000-8000 字 | 昨夜全球市场、今日 A/H 股机会和风险 |
| 午间 | 午盘快报 | 1500-3000 字 | 上午异动、资金流、下午策略 |
| 收盘后 | 收盘复盘与明日展望 | 3000-5000 字 | 全天复盘、公告解读、明日策略 |

默认时间建议：

```text
08:00 盘前日报
12:30 午盘快报
17:30 收盘复盘
```

具体时间后续可按 A 股交易日历调整。

## 7. 数据处理流程

```text
数据抓取
  ↓
原始数据保存
  ↓
清洗正文
  ↓
去重与聚类
  ↓
实体识别：公司、行业、国家、资产
  ↓
事件重要性评分
  ↓
来源可信度评分
  ↓
AI 摘要与事实校验
  ↓
投研分析
  ↓
生成结构化报告 JSON
  ↓
生成网页、推送、保存历史
```

## 8. 事件评分体系

每条事件打 0-100 分。

评分维度：

| 维度 | 说明 |
|---|---|
| 影响范围 | 全球、国家、行业、公司 |
| 市场敏感度 | 是否影响股价、利率、汇率、商品 |
| 时间敏感度 | 是否必须今天处理 |
| 来源可信度 | 官方、主流媒体、二手转述、传闻 |
| 产业相关性 | 是否与 AI、新能源、A/H 股主线相关 |
| 公司重要性 | 是否涉及龙头、指数权重、产业链核心 |
| 预期差 | 是否超出市场已有预期 |
| 持续性 | 短期情绪还是中长期趋势 |

评分结果：

| 分数 | 处理方式 |
|---:|---|
| 85-100 | 进入十大核心事件候选 |
| 70-84 | 进入行业或公司重点分析 |
| 50-69 | 作为辅助信息 |
| 0-49 | 存档，不进入正文或仅简略提及 |

## 9. 投资建议体系

系统可以给出明确研究性行动建议，但必须附带逻辑、置信度、风险和失效条件。

### 9.1 建议动作

| 动作 | 含义 |
|---|---|
| 可分批关注 | 逻辑较强，价格或趋势较匹配 |
| 持有观察 | 已有仓位可继续观察，暂不加仓 |
| 事件驱动可参与 | 短期催化明显，持续性需验证 |
| 谨慎追高 | 逻辑成立，但短期拥挤或涨幅较大 |
| 等待确认 | 有潜在机会，但数据或价格信号不足 |
| 降低关注 | 逻辑减弱或风险上升 |
| 回避 | 风险收益比差，暂不参与 |

### 9.2 建议字段

每个投资建议必须包含：

- 建议动作
- 投资周期：1 周、1 个月、3 个月、1 年
- 成功率判断
- 置信度
- 风险等级
- 核心逻辑
- 核心催化剂
- 代表行业、ETF、公司
- 是否已被市场定价
- 适合仓位：轻仓、标准、偏重
- 失效条件
- 持续观察指标

### 9.3 成功率解释

成功率不是承诺收益，而是系统基于事实、趋势、估值、资金和风险后的研究判断。

| 成功率 | 含义 |
|---:|---|
| 75% 以上 | 多项因素共振，但仍需风险控制 |
| 60%-75% | 中等偏高，适合重点关注 |
| 50%-60% | 有机会，但不确定性较高 |
| 50% 以下 | 暂不建议参与 |

## 11. AI 分析链路

第一版采用单程序、多阶段提示词链路，不一开始做复杂多智能体服务。

逻辑角色包括：

| 角色 | 任务 |
|---|---|
| 新闻筛选代理 | 筛选过去 24 小时最重要信息 |
| 宏观代理 | 分析利率、汇率、流动性、政策 |
| A 股代理 | 分析 A 股板块、公告、主题 |
| 港股代理 | 分析港股科技、南向资金、估值 |
| 美股代理 | 分析科技、AI、半导体和美债影响 |
| AI 行业代理 | 分析模型、算力、芯片、云、应用 |
| 新能源代理 | 分析电池、储能、光伏、风电、电网 |
| 风险代理 | 识别地缘、政策、流动性、估值风险 |
| 基金经理代理 | 给出配置与行动建议 |
| 校对代理 | 检查事实、逻辑跳跃、重复和不确定来源 |
| 主编代理 | 统一成最终中文报告 |

## 14. 推送设计

### 14.1 PushPlus

PushPlus 用于向个人微信发送提醒。系统只发送摘要和链接，不发送全文。

示例：

```text
【AI 投研日报 - 盘前版】

市场判断：中性偏积极
今日主线：AI 算力、港股科技、新能源电网
风险等级：中等

今日最值得关注：
1. xxx
2. xxx
3. xxx

完整报告：
https://your-site.github.io/ai-research/?token=xxx
```

### 14.2 邮件备份

邮件用于防止微信推送失效。

内容包括：

- 报告标题
- 核心结论
- 今日行动清单
- 完整网页链接

## 15. GitHub Actions 自动任务

定时任务包括：

```text
morning_report.yml
noon_report.yml
close_report.yml
```

每个任务执行：

1. 拉取代码
2. 安装依赖
3. 读取环境变量
4. 抓取数据
5. 生成报告
6. 生成网页
7. 提交报告文件
8. 发布 GitHub Pages
9. 发送 PushPlus 和邮件

## 20. 核心原则

系统必须长期遵守以下原则：

1. 事实、预期、判断必须区分。
2. 所有重要观点必须有来源或逻辑支撑。
3. 不把单一传闻作为投资依据。
4. 每个投资建议必须包含风险和失效条件。
5. 报告重分析，不堆砌新闻。
6. A 股和港股优先，美股和全球宏观用于解释资金环境。
7. 系统应能在部分数据源失败时降级运行。
8. 用户配置应尽量少，自动化优先。
